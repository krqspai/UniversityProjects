Task Description

You are tasked with helping a company innovation manager with handling an innovation competition. The ranking of employees in the competition is carried out based on the number of innovation proposals and implemented innovations.

Your task is to write two classes: Employee and InnovationCompetition. The classes should be in the hr.fer.oop package and have visibility set to public.

It is necessary to implement the methods in the classes as shown in the class diagram. You are free to add and implement additional methods, attributes and constructors.

Visibility legend in the class diagram:

You must prepare the Employee class to work with collections by overriding the equals and hashCode methods.

The equals method returns that two employees are equal if they have the same first and last name. Likewise, the hashCode method will return the same hash for two employees who have the same first and last name.

You also need to rearrange the toString method to match the output shown in the example.

The addEmployee method in the InnovationCompetition class enables adding of an employee in the company's innovation competition. Employees can be removed from the competition using the removeEmployees method which has as a parameter the list of last names of employees that we want to exempt.

In order to rank the existing list of employees in the competition by the rankEmployees method, it is necessary to write a class that implements the Comparator interface and compares two employess first by the implementedInnovations attribute (higher goes first), then by the proposedInnovations attribute (higher goes first) and finally by last name (alphabetical).
